java反序列化

本文最后更新于 2021年4月5日 中午

介绍

java 反序列化是指把 java对象转换为字节序列的过程，java反序列化则是指把字节序列恢复为java对象的过程。

为什么需要这个功能？在通信场景需要通过序列化数据进行对象传送，最常用的场景即跨机器调用通信，需保持对象是一致的。

常见的序列化协议：

  • XML SOAP
  • JSON
  • Protobuf

序列化注意：

  • 1、在java中只有实现了 Serializable 或者 Externalizable 接口类（implements Serializable）的对象才能被序列化为字节序列。
  • 2、静态成员变量不能被反序列化，因为它属于类
  • 3、transient 标识的对象成员变量不参与序列化，进一步常配合重写readObject/writeObject方法：
  • 4、writeObject 和 readObject 是一一对应关系，写了一个对象，相应就可读到一个对象

自动调用readObject机制

以HashMap为例子，先让我们了解一下是类往往是如何重写readObject方法的：

  • **<font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">s.defaultReadObject();</font>**：这一步会自动反序列化所有非 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">transient</font> 和非 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">static</font> 字段。对于 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">HashMap</font> 来说，这些字段包括 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">loadFactor</font>（负载因子）和 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">threshold</font>（阈值）。注意，<font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">table</font> 是 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">transient</font> 的，所以不会被自动反序列化。
  • **<font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">reinitialize();</font>**：这是一个自定义的方法，用于重新初始化一些内部状态，比如清空 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">modCount</font> 等。这确保了反序列化后的 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">HashMap</font> 处于一个干净的状态。
  • 这里是对 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">loadFactor</font> 的有效性检查，确保它是正数且不是 NaN（非数字）。
  • **<font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">s.readInt();</font>**：这里读取了一个 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">int</font> 值，表示 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">HashMap</font> 在序列化时的桶的数量（buckets）。但是，在反序列化过程中，这个值被忽略了，因为它只是为了兼容性目的而保留的。
  • **<font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">int mappings = s.readInt();</font>**：接着读取了另一个 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">int</font> 值，表示 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">HashMap</font> 中实际存储的键值对数量（即 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">size</font>）。这个值非常重要，因为它决定了需要从字节流中读取多少个键值对来重建 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">HashMap</font>。
  • 容量计算：根据 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">mappings</font> 和 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">loadFactor</font> 计算出合适的初始容量 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">cap</font> 和阈值 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">threshold</font>。这里的逻辑是为了确保 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">HashMap</font> 的容量既不过小也不过大，从而保证性能。
  • 数组创建：使用 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">SharedSecrets.getJavaOISAccess().checkArray(s, Map.Entry[].class, cap);</font> 检查是否允许创建指定大小的数组，然后创建一个新的 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">Node<K,V>[]</font> 数组作为 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">table</font>。
  • 读取键值对：接下来是一个循环，读取并插入 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">mappings</font> 个键值对。每个键值对都是通过 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">s.readObject()</font> 读取出来的，并使用 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">putVal()</font> 方法插入到 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">HashMap</font> 中。

事实上重写的readObject方法不可能直接含有危险方法调用，那么就需要寻找可能的调用链来帮助执行恶意代码。

可能的形式如下：

  • 入口类参数中包含可控类，该类存在危险方法），readObject时调用
  • 入口类参数中包含可控类，该类又调用其他有危险方法的类，readObject时调用
  • 链式调用中重点是找重写的相同类型的同名函数 (如 Object有：hashCode、equals、toString )
  • 构造函数、静态代码块等类加载时隐式执行

反射

这里需要提到反射的概念：

  • java可通过反射改变已有对象的属性，事先并不知道类对象是谁（场景大多是事先已知方法或者属性），进而可以进行动态调用方法、生成对象、操作内部类和私有方法等行为。
  • <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">Class</font> 类是一个特殊的类，用来表示运行时的类信息。每个加载到 JVM 中的类都有一个对应的 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">Class</font> 实例，这个实例包含了该类的所有结构信息（如字段、方法、构造函数等）。
  • 没有继承关系，每个类都有一个对应的 Class 实例，反射就是操作 Class 对象

反射的作用：

  • 修改方法、属性，定制需要的对象
  • 通过invoke调用除了同名函数以外的函数
  • 通过Class类创建对象，引入不能反序列化的类

动态代理

动态代理（Dynamic Proxy） 是 Java 提供的一种在运行时动态创建代理类的机制。它允许你为一个或多个接口生成一个“代理类”，这个代理类可以在调用目标方法前后插入自定义逻辑。

在不修改目标对象的前提下，通过代理对象对方法进行增强（如日志记录、权限控制、事务管理等）。

Java 的动态代理主要依赖于两个类/接口：

  • <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">java.lang.reflect.Proxy</font>：用于生成代理类。
  • <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">java.lang.reflect.InvocationHandler</font>：负责拦截代理类的方法调用，并决定如何处理。

动态代理是一种设计模式，可以在不修改原有类上增加功能，如果是静态代理，你每一个方法都要重写，很麻烦，用动态则修改的代码少，适配强。

一句话概括

在反序列化时，入口类重写了readObject方法，方法内部会调用入口类相应对象的一些常用方法（如key.hashCode()），进一步通过寻找利用类（其存在危险操作的同名函数，如B.hashCode()），结合反射、动态代理、类加载机制等制作调用链，从而在服务器上执行恶意代码

✅ 修复

  • 不要反序列化不可信的数据。
  • 使用 <font style="color:rgb(44, 44, 54);background-color:rgba(175, 184, 193, 0.2);">ObjectInputFilter</font> 来限制允许反序列化的类。
  • 升级到安全版本（如 Commons Collections 4.x）。
  • 尽量避免使用 Java 原生序列化，改用 JSON、XML 等更安全的格式。
